# pnpm symlink resolution issue - demo

## The Problem

**Before deployment (pnpm structure):**
```
node_modules/
  sea-turtle/ → symlink to .pnpm/sea-turtle@1.0.0/node_modules/sea-turtle/
  .pnpm/
    sea-turtle@1.0.0/
      node_modules/
        sea-turtle/
          index.js              ← requires('sea-turtle-shell')
        sea-turtle-shell/ → symlink to ../../sea-turtle-shell@1.0.0/...
```

When `sea-turtle/index.js` requires `sea-turtle-shell`, Node finds it right next door at `../sea-turtle-shell/`.

**After deployment (dereferenced):**
```
node_modules/
  sea-turtle/
    index.js    ← requires('sea-turtle-shell')... but where is it?
```

Dereferencing flattened `sea-turtle/` into a standalone directory. The sibling `sea-turtle-shell/` is gone. Node walks up looking for `node_modules/sea-turtle-shell` — **doesn't exist → crash**.
